# Code of Conduct

## Our pledge

This project is intended to be a practical, respectful, and useful space for people experimenting with NGINX, OpenSSL, Docker, DNS, TLS, and Encrypted ClientHello.

Participants are expected to communicate in a way that is constructive, professional, and focused on improving the project.

## Expected behavior

Examples of expected behavior include:

- Being respectful of differing skill levels and backgrounds
- Asking clear technical questions
- Giving constructive feedback
- Explaining tradeoffs and assumptions
- Reporting bugs with enough detail to reproduce them
- Keeping discussions focused on the project
- Respecting privacy, security, and responsible disclosure boundaries

## Unacceptable behavior

Examples of unacceptable behavior include:

- Harassment, threats, or personal attacks
- Insults, trolling, or deliberately inflammatory comments
- Publishing someone else's private information without permission
- Sharing secrets, credentials, private keys, tokens, or sensitive logs
- Encouraging unsafe security practices without clearly identifying the risk
- Repeated off-topic or disruptive comments
- Any conduct that would make reasonable contributors feel unwelcome or unsafe

## Security-sensitive content

This project deals with TLS, DNS, ECH configuration, Docker, and potentially internet-facing infrastructure.

Do not post:

- API tokens
- Private keys
- TLS private key material
- ECH private/generated key material
- Real production secrets
- Real internal hostnames or private IP mappings unless intentionally disclosed
- Logs containing sensitive headers, tokens, or credentials

If you believe you found a security issue, follow the guidance in `SECURITY.md`.

## Scope

This code of conduct applies to:

- GitHub issues
- Pull requests
- Discussions
- Code review
- Project documentation
- Any other project-related communication

## Enforcement

The project maintainer may remove, edit, or reject comments, commits, issues, pull requests, or other contributions that do not align with this code of conduct.

The maintainer may also temporarily or permanently block users from participating in the project if needed to protect the project and its contributors.

## Reporting

If you need to report conduct concerns, contact the project maintainer privately through the maintainer's GitHub profile or another listed project contact method.

When reporting, include:

- A description of the issue
- Links to relevant comments, issues, or pull requests
- Any relevant context
- Whether the concern involves security-sensitive information

## Maintainer note

This is a small public lab project. The goal is to keep the project useful, respectful, and safe for people who want to learn, test, adapt, and improve direct-origin ECH patterns.
